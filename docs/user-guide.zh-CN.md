# CoreRobin 使用指南

CoreRobin 把 CPU、内存、磁盘、网络和正在运行的应用放在一个清晰的桌面界面里。电脑变慢、磁盘快满或网络突然繁忙时，你可以用它更快找到原因。

## 下载与安装

从 [GitHub Releases](https://github.com/JimmyDaddy/corerobin-monitor/releases/latest) 下载适合当前系统的安装包。当前 macOS DMG 已使用 Developer ID 签名、经过 Apple 公证并装订票据；Gatekeeper 会在发布前再次验证。Apple Silicon 是主要实机验证路径，Intel 包完成同等签名与自动校验，但尚未分别完成 Intel 实机验收。Windows 与 Linux 安装包目前属于早期预览版本，尚未配置平台签名，也未分别完成真实设备验证。Release 会同时提供 SHA-256 校验表、SPDX SBOM，以及校验表的 Sigstore 签名包；这些来源完整性记录不能代替平台签名或真实设备验证。

## 第一次打开

1. 先等几秒，让第一轮数据准备好。刚启动时，网络和磁盘速度可能显示“预热”；下一次刷新后就会出现。
2. 日常模式首页会先告诉你电脑现在怎么样，并给出一个最值得做的动作。没有问题时，不需要逐项检查卡片。
3. 如果你已经感觉到卡顿、发热、耗电、空间不足、开机慢或网络异常，请打开“帮我解决”，选择最接近的情况。
4. 右上角的设置可以更改语言、通知、历史记录和后台启动方式。界面支持简体中文、繁體中文、English、日本語、Deutsch、Français、Español、Português (Brasil)、한국어和Русский。语言与日常/专业模式使用一致的顶部控件；需要查看进程树、连接或命令行时，可以切换到专业模式。

## 主要页面

### 电脑状态

- Robin 状态伙伴会给出一个当前结论和一个主动作。没有持续问题时，它会直接说明现在不用处理任何事。
- 有问题时，主动作会打开最优先的问题，不会让你先在 CPU、内存、磁盘和网络之间做选择。
- 首页只保留一件最值得留意的事和一条最近变化。CoreRobin 不会因为一次短暂波动就说电脑出了问题，也不会用没有依据的分数评价电脑。
- 持续问题确认后会获得稳定身份，不会因为一次正常采样突然消失。指标回落后会先显示“正在确认恢复”，稳定恢复后再退出待处理数量。
- 暂时无法读取的数据会明确显示为不可用，不会被当作正常。

### 帮我解决

- 从“电脑有点卡”“风扇一直很响”“电池掉得很快”“空间快不够了”“开机越来越慢”或“网络好像有问题”开始。
- 页面只展示当前阶段：正在观察、发现原因、选择处理或重新检查，不再要求你理解固定的技术步骤。
- 每个结果只说明发生了什么、为什么值得留意、现在可以做什么。CPU、内存、磁盘等依据收在“为什么这样判断”中。
- 请求应用退出、停用启动项或删除文件前都需要你确认；CoreRobin 不会自动替你执行。

### 应用

- “应用”是日常和专业模式中的一级入口。在日常模式中，它提供稳定的应用影响快照；专业模式会直接打开“彻底卸载助手”。
- 应用清单优先显示与界面语言匹配的本地化名称，并复用应用自身图标。清单会在本机缓存并根据应用目录指纹后台刷新，重复进入时不必每次完整扫描；手动刷新会强制重新读取。
- “彻底卸载助手”目前支持 macOS 顶层应用。它先识别应用本体和由 Bundle ID 明确归属的残留路径，再让你逐项复核；默认尝试移到废纸篓，直接删除仍需单独确认。CoreRobin 不会卸载自身，也不会处理无法安全归属的路径。
- CPU、内存和磁盘数据收在“为什么这样判断”中。
- 专业模式提供进程树、搜索、排序、路径、启动命令和最近五分钟的资源变化。
- 结束进程时，优先选择“请求结束”，让应用有机会保存内容。只有应用完全没有响应时，再考虑强制结束。
- 为了避免结束错进程，CoreRobin 会在执行前再次检查目标。如果进程已经退出或发生变化，操作会停止。
- 操作完成后，日常模式会重新检查应用是否仍在运行，并明确告诉你结果。
- 系统关键进程和 CoreRobin 本身不能被误关。

### 存储

- 查看每个磁盘还剩多少空间、最近的读写变化，以及哪些应用正在频繁读写。
- 磁盘空间不足时，可以从这里直接进入“清理”页面。

### 清理

清理页面在首次进入且尚无结果时，会先显示扫描范围、只读边界和“开始只读扫描”按钮；开始后再展示实时扫描进度。扫描完成后用扇形图显示空间都被什么占用了。默认的“文件路径”视图严格按照真实目录层级展示；“用途分类”视图则根据常见位置汇总下载、缓存和应用数据，只作为辅助理解。扫描不会进入外置磁盘或其他挂载点。

在 macOS 上，邮件、信息、其他应用数据等位置受系统保护。首次开始扫描时，CoreRobin 会先说明“完全磁盘访问权限”的用途，并打开“系统设置 → 隐私与安全性 → 完全磁盘访问权限”。如果列表里还没有 CoreRobin，请点击列表下方的 `+`，选择 `CoreRobin.app`；应用内的“在访达中显示应用”可以帮你快速定位。开启后返回应用，扫描会自动继续；也可以暂不授权，只扫描当前可访问的位置。这个权限始终由 macOS 管理，可以随时关闭。

> 只删除你认识、并且确定不再需要的内容。不确定的内容先保留。可以重新生成的缓存适合作为起点，但缓存也不等于全部都能安全删除。

1. **等待扫描完成**：页面会持续显示正在扫描的位置、已经检查的项目和找到的空间。扫描会一直进行到完成，除非你主动停止。
2. **查看大文件夹**：扇区越大，占用的空间越多。单击文件夹进入，单击中间区域返回上一级。
3. **放进清理篮**：按住扇区或右侧目录列表中的项目开始拖动。扫描结果存在时，清理篮会固定显示在窗口底部；放进清理篮不会立即移动或删除文件。
4. **刷新并选择清理方式**：CoreRobin 会只重扫清理篮中的目标并显示最新大小和文件数。确认信息正确后，可以选择“移到废纸篓”（推荐，可在清空废纸篓前恢复）或“直接删除”（不可撤销）。

完整扫描结果会在本机保留最多 7 天，方便继续浏览；超过 24 小时会明确标记为较旧，涉及清理前建议先更新。缓存或过期结果不能直接作为清理依据：打开清理复核时，CoreRobin 会只对清理篮中的目标做权威重扫，并把本次确认绑定到所选清理方式。如果大小、文件数或目录内容发生变化，旧确认会失效；看到更新后的结果并再次刷新稳定后，必须重新勾选确认。其他位置发生较多变化时，请选择“重新扫描”。

“重复与长期未修改文件”是独立的主动检查：它只扫描桌面、文稿、下载、影片、音乐和图片，不跟随符号链接；先按大小筛选候选，再读取可能重复的文件并在本机计算 SHA-256。长期未用按“180 天未修改且超过 100 MB”近似判断，因为多数平台不能可靠提供最后访问时间。结果只提供预览、复制路径和在文件管理器中显示，不会自动删除。

#### 清理前务必知道

- “移到废纸篓”在系统废纸篓清空前通常可以恢复；“直接删除”不会进入废纸篓，也无法通过 CoreRobin 恢复。
- 如果 CoreRobin 无法安全打开系统废纸篓，会说明失败原因并允许重试；只有你主动改选“直接删除”并再次确认后，才会进入不可恢复流程。
- 建议先清理可以重新生成的缓存。下载内容、项目文件、设置和个人资料不确定时不要删除。
- CoreRobin 只处理你个人文件夹内的普通文件和文件夹。
- 主目录、废纸篓本身、链接、特殊文件和其他磁盘不会被删除。
- 真正清理前，应用会再次核对整棵目标树。发现深层新增文件或其他不一致时会停止操作，并要求重新刷新、确认。
- 清理成功的项目会立即从扇形图和清理篮中消失；失败的项目会保留下来并说明原因。

### 网络

- 网络不再是日常模式的一级页面；选择“网络好像有问题”后，CoreRobin 会先检查当前连接和收发状态，再给出结论。
- 查看当前上传和下载速度、本次启动后的累计流量、网络接口和活动连接。
- 打开网络页面会自动检查一次 DNS、TCP 连接延迟、抖动、估算丢包和联网状态；停留期间每 30 秒进行一次轻量采样，形成最近 15 分钟趋势，并显示平均值、峰值和异常次数。离开页面后停止采样，也可以随时“立即复测”。
- 连接历史默认关闭。单独启用后，会在本机保存五分钟级应用、主机名或 IP 聚合，最多保留 1、7 或 30 天并限制为 5000 条；可随时清空。
- 可以按 TCP、UDP 和连接状态筛选列表。
- 系统不一定允许普通应用读取每条连接属于哪个进程，所以没有显示应用名并不代表连接有问题。

### 启动项

- 启动项不再是日常模式的一级页面；选择“开机越来越慢”后才会出现相关检查和可恢复的操作。
- 查看哪些应用会随电脑启动，以及它们来自哪里。
- 支持的第三方项目可以关闭，也可以稍后恢复。
- 系统项目和当前平台无法安全修改的项目只供查看。
- CoreRobin 不会直接删除启动项的配置文件。
- 如果 CoreRobin 确实由系统登录项以后台模式启动，会在最多 3 分钟内测量 CPU、磁盘和应用峰值及趋稳时间；手动打开应用不会被误记为登录启动影响。

### 历史

- 日常模式的“记录”用简单时间线说明电脑什么时候开始变忙、什么时候恢复正常。专业模式保留完整的资源筛选和技术历史。
- 同一次问题的提醒与恢复会合并为一条事件，保留开始时间、持续时长、可能原因和最终结果。
- 你确认执行的应用退出或重启、永久清理和启动项操作也会记录执行状态与复核结果；永久清理只保存项目数量和释放空间，不保存文件路径。
- 记录可以保留 1、7 或 30 天，也可以关闭或随时清空。
- 只有问题持续一段时间时才会提醒，同一问题不会反复弹出。
- 问题稳定恢复后会补充一次恢复提醒；主窗口、状态栏面板和 Robin 伙伴使用同一份健康状态。
- CPU、内存和磁盘提醒可以分别关闭。
- 应用和启动项名称只有在你允许保存名称后才会跨重启保留；不保存命令行、文件路径、文件名或连接地址。

### 设置

- 设置按界面、后台与桌面、采样与视图、历史与提醒等主题分组，卡片保持一致的网格与控件规格；日常模式提供语言、文字大小、动态效果、桌面提醒、本机历史和 Robin 伙伴偏好。
- 可以让 CoreRobin 在登录系统后静默启动，不打开主窗口；macOS 还可以选择是否在 Dock 和应用切换器中显示图标。
- 专业模式的设置中还可以调整采样速度、连接刷新速度、颜色提示和默认进程视图。首次进入进程页时会自动选择一个适合检查的高负载进程，右侧详情不再保持空白。
- 专业模式可通过与界面一致的应用选择、指标、阈值和持续时间控件，为指定应用设置 CPU、内存或磁盘持续阈值。条件达到后发送一次桌面提醒，恢复后才允许再次提醒，并有 10 分钟冷却。
- 专业概览会在平台支持时显示 GPU 使用率和应用能耗相对影响；相对影响不是瓦特或电量，当前能力以 macOS 为主，其他平台无法读取时明确显示不可用。

## 状态栏面板

关闭主窗口后，CoreRobin 仍会留在状态栏。单击状态栏图标会打开与主窗口共享同一健康结论的紧凑面板；再次单击会收起，双击会直接打开主窗口。登录时静默启动不会弹出启动画面或主窗口。

选择“Robin 伙伴”会在桌面右下角打开一个紧凑小窗口。它与主窗口、状态栏面板共享当前结论，只显示一个下一步；按 Esc、点击关闭按钮或点到其他窗口都会收起。拖动后的位置会保留，也可以在日常助手设置中选择是否置顶或随应用启动显示。

如果要完全停止监控，请退出 CoreRobin，而不是只关闭窗口。

## 平台支持与已知限制

| 平台 | 当前状态 | 已验证范围 |
| --- | --- | --- |
| macOS · Apple Silicon | 推荐 | M 系列 Mac 是主要实机验证路径；提供 `aarch64.dmg` |
| macOS · Intel | 可用，尚待实机验收 | 提供已签名、公证并装订票据的 `x64.dmg`；自动构建、架构、签名和 Gatekeeper 校验通过，但未完成 Intel 实机验收 |
| Windows · x64 | 早期预览 | 自动构建与测试通过；真实设备、系统集成与安装器覆盖仍有限 |
| Linux · x64 | 早期预览 | 自动构建与测试通过；不同发行版、桌面环境和 WebKitGTK 组合尚未全面验证 |

当前 macOS 安装包已配置 Developer ID 签名与 Apple 公证；Windows 安装包尚无平台签名，Windows 与 Linux 仍是早期预览。温度、电池健康度、连接归属和部分启动项能力依赖操作系统与硬件，无法读取时会显示为不可用。网站只根据浏览器推荐操作系统，无法可靠判断 Mac 芯片，也不会自动替你下载。每个版本的变化和当前限制见[公开 Release Notes](https://monitor-app.corerobin.com/releases/)。

## 常见问题

### 刚启动时为什么没有网络或磁盘速度？

应用需要先后读取两次数据才能算出速度。等到下一次刷新就会显示。

### 为什么扫描要等很久？

扫描时间主要取决于文件数量和磁盘速度，而不是磁盘总容量。大量小文件通常比少量大文件更慢。页面会持续更新进度，你也可以主动停止。

### 为什么大小和 Finder 或 Explorer 显示的不一样？

CoreRobin 显示文件实际占用的磁盘空间。压缩文件、稀疏文件和硬链接等情况，可能会让这个数字与文件标称大小不同。

### 为什么有些连接没有应用名？

操作系统可能不允许普通应用读取连接属于哪个进程。CoreRobin 不会为了补全这项信息自动申请更高权限。

### 我应该下载哪个安装包？

在下载页按系统与芯片选择：所有 Apple Silicon Mac（包括 M3 等 M 系列芯片）下载 `aarch64.dmg`，只有 Intel Mac 才下载 `x64.dmg`。网站可以识别 macOS，但无法可靠判断 Mac 芯片；不确定时请打开“ → 关于本机”查看。Windows 选择 `x64-setup.exe` 或 `x64_en-US.msi`；Linux x64 可选 `AppImage` 或 `deb`。Windows 与 Linux 目前是早期预览版本。下载页会说明平台状态、文件大小和校验方式。

### macOS 提示无法验证 CoreRobin，或“完全磁盘访问权限”列表里没有它，怎么办？

当前 macOS 安装包已经 Apple 公证并装订票据，最新版通常不需要使用“仍要打开”。如果仍被拦截，请确认应用来自官方 Release、安装在“应用程序”文件夹，并核对 SHA-256；问题仍存在时请报告 macOS、芯片和 CoreRobin 版本。完全磁盘访问权限列表中没有 CoreRobin 时，请先从“应用程序”启动一次，再在列表下方点击 `+` 选择它；应用内也可以在访达中显示应用位置。

### macOS 提示 CoreRobin“已损坏”，要用“仍要打开”吗？

不要。“已损坏”表示包体、签名或下载完整性异常，不应使用“仍要打开”或移除隔离属性绕过。v0.0.3 的 Apple Silicon DMG 曾存在签名不完整问题，已由后续版本替换；当前 macOS 包还会经过 Developer ID、Apple 公证、票据装订和 Gatekeeper 校验。请删除旧 DMG 和旧应用副本，重新下载最新版并用 `SHA256SUMS` 核对文件；如果仍出现相同提示，请报告 macOS 版本、Mac 芯片、CoreRobin 版本和校验结果。

### 不授予完全磁盘访问权限，可以扫描吗？

可以。选择“扫描可访问区域”后，CoreRobin 会继续读取当前账户和系统允许访问的位置，并把受保护或暂时无法读取的位置明确标记出来。授权只用于尽可能完整地查看系统磁盘，不改变扫描只读取文件元数据的边界。

### 关闭主窗口后，CoreRobin 和 Robin 伙伴还在运行吗？

会。关闭主窗口只会隐藏窗口，监控会继续在状态栏运行。要停止监控，请从状态栏菜单退出 CoreRobin。Robin 伙伴可用关闭按钮、Esc 或设置中的开关收起；它不等于退出应用。

### 如何清空记录，或完整卸载？

1. 在设置中关闭“登录后启动”。如果准备继续使用应用，只想重置数据，可以在“设置 → 关于与支持”选择“清除全部本机数据”。
2. 也可以分别在“记录”页清空历史、在清理页清除已保存的扫描结果。
3. 在 macOS 的“系统设置 → 隐私与安全性 → 完全磁盘访问权限”中关闭 CoreRobin。
4. 从状态栏面板选择“退出应用”，然后在活动监视器、任务管理器或系统监视器中确认没有 CoreRobin 进程。
5. 移除 CoreRobin 应用。彻底卸载助手不能卸载正在运行的 CoreRobin 自身；需要手动核对残留时，请参考[隐私说明中的各平台数据位置](privacy.zh-CN.md#完整卸载与本机数据位置)。

不要把日志、本机数据目录、未隐藏隐私信息的截图或自动生成的诊断内容直接提交到公开 Issue。反馈前请在“设置 → 关于与支持”复制版本与诊断摘要，预览并移除用户名、路径、IP 地址等信息。

## 隐私与问题反馈

监控、文件、进程、历史和连接信息只保存在这台电脑上，不会上传或同步。常规空间扫描只读取名称、大小、位置等元数据；只有你主动开始重复文件检查时，才会读取候选文件内容并在本机计算哈希。数据保留、完全磁盘访问权限和清除方式请参阅[隐私说明](privacy.zh-CN.md)。

普通问题可以在 GitHub Issues 中反馈。请从“设置 → 关于与支持”复制版本与系统信息，并写清楚刚才做了什么以及看到的错误。发布变化见[Release Notes](https://monitor-app.corerobin.com/releases/)；安全问题请使用 GitHub 的私密漏洞报告功能。
