# CoreRobin 使用指南

CoreRobin 把 CPU、内存、磁盘、网络和正在运行的应用放在一个清晰的桌面界面里。电脑变慢、磁盘快满或网络突然繁忙时，你可以用它更快找到原因。

## 下载与安装

从 [GitHub Releases](https://github.com/JimmyDaddy/corerobin-monitor/releases/latest) 下载适合当前系统的安装包。当前发布构建尚未配置 Developer ID、Apple 公证或 Windows 平台签名。macOS 版本已在真实设备上验证；如果系统阻止第一次打开，请前往“系统设置 → 隐私与安全性”，确认打开 CoreRobin。Windows 与 Linux 安装包目前属于早期预览版本。Release 会同时提供 SHA-256 校验表、SPDX SBOM 和 GitHub artifact provenance；这些来源完整性记录不能代替平台签名。

## 第一次打开

1. 先等几秒，让第一轮数据准备好。刚启动时，网络和磁盘速度可能显示“预热”；下一次刷新后就会出现。
2. 日常模式首页会先告诉你电脑现在怎么样，并给出一个最值得做的动作。没有问题时，不需要逐项检查卡片。
3. 如果你已经感觉到卡顿、发热、耗电、空间不足、开机慢或网络异常，请打开“帮我解决”，选择最接近的情况。
4. 右上角的设置可以更改语言、通知、历史记录和后台启动方式。界面支持简体中文、繁體中文、English、日本語、Deutsch、Français、Español、Português (Brasil)、한국어和Русский。需要查看进程树、连接或命令行时，可以使用页面顶部的“进入专业模式”。

## 主要页面

### 电脑状态

- Robin 状态伙伴会给出一个当前结论和一个主动作。没有持续问题时，它会直接说明现在不用处理任何事。
- 有问题时，主动作会打开最优先的问题，不会让你先在 CPU、内存、磁盘和网络之间做选择。
- 首页只保留一件最值得留意的事和一条最近变化。CoreRobin 不会因为一次短暂波动就说电脑出了问题，也不会用没有依据的分数评价电脑。
- 持续问题确认后会获得稳定身份，不会因为一次正常采样突然消失。指标回落后会先显示“正在确认恢复”，稳定恢复后再退出待处理数量。
- 暂时无法读取的数据会明确显示为不可用，不会被当作正常。

### 帮我解决

- 从“电脑有点卡”“风扇一直很响”“电池掉得很快”“空间快不够了”“开机越来越慢”或“网络好像有问题”开始。
- 页面只展示当前阶段：正在观察、发现原因、选择处理或重新检查，不再要求你理解固定的技术步骤。
- 每个结果只说明发生了什么、为什么值得留意、现在可以做什么。CPU、内存、磁盘等依据收在“为什么这样判断”中。
- 请求应用退出、停用启动项或删除文件前都需要你确认；CoreRobin 不会自动替你执行。

### 应用

- 应用不再是日常模式的一级页面。只有检查结果与应用有关时，才会显示一份稳定快照。
- 快照会把同一应用的相关进程合在一起，并先说明它对电脑的影响。列表不会每秒重新排序；需要时可以手动更新或展开全部应用。
- CPU、内存和磁盘数据收在“为什么这样判断”中。
- 专业模式提供进程树、搜索、排序、路径、启动命令和最近五分钟的资源变化。
- 结束进程时，优先选择“请求结束”，让应用有机会保存内容。只有应用完全没有响应时，再考虑强制结束。
- 为了避免结束错进程，CoreRobin 会在执行前再次检查目标。如果进程已经退出或发生变化，操作会停止。
- 操作完成后，日常模式会重新检查应用是否仍在运行，并明确告诉你结果。
- 系统关键进程和 CoreRobin 本身不能被误关。

### 存储

- 查看每个磁盘还剩多少空间、最近的读写变化，以及哪些应用正在频繁读写。
- 在 macOS 上，属于同一个系统卷的 `/` 和 Data 卷会合并显示，不会重复计算。
- 磁盘空间不足时，可以从这里直接进入“清理”页面。

### 清理

清理页面会完整扫描系统磁盘，并用扇形图显示空间都被什么占用了。默认的“文件路径”视图严格按照真实目录层级展示；“用途分类”视图则根据常见位置汇总下载、缓存和应用数据，只作为辅助理解。扫描不会进入外置磁盘或其他挂载点。

在 macOS 上，邮件、信息、其他应用数据等位置受系统保护。首次开始扫描时，CoreRobin 会先说明“完全磁盘访问权限”的用途，并打开“系统设置 → 隐私与安全性 → 完全磁盘访问权限”。如果列表里还没有 CoreRobin，请点击列表下方的 `+`，选择 `CoreRobin.app`；应用内的“在访达中显示应用”可以帮你快速定位。开启后返回应用，扫描会自动继续；也可以暂不授权，只扫描当前可访问的位置。这个权限始终由 macOS 管理，可以随时关闭。

> 只删除你认识、并且确定不再需要的内容。不确定的内容先保留。可以重新生成的缓存适合作为起点，但缓存也不等于全部都能安全删除。

1. **等待扫描完成**：页面会持续显示正在扫描的位置、已经检查的项目和找到的空间。扫描会一直进行到完成，除非你主动停止。
2. **查看大文件夹**：扇区越大，占用的空间越多。单击文件夹进入，单击中间区域返回上一级。
3. **放进清理篮**：按住扇区开始拖动，清理篮会固定显示在窗口底部。放进清理篮不会立即删除文件。
4. **刷新后永久删除**：CoreRobin 会只重扫清理篮中的目标并显示最新大小和文件数；确认这些信息正确后再继续。

完整扫描结果会在本机保留 7 天，返回清理页面时可以继续浏览。缓存或过期结果不能直接作为永久删除依据：打开删除复核时，CoreRobin 会只对清理篮中的目标做权威重扫。如果大小、文件数或目录内容发生变化，旧确认会失效；看到更新后的结果并再次刷新稳定后，必须重新勾选确认。其他位置发生较多变化时，请选择“重新扫描”。

#### 删除前务必知道

- 文件不会进入系统废纸篓，删除后无法通过 CoreRobin 恢复。
- 建议先清理可以重新生成的缓存。下载内容、项目文件、设置和个人资料不确定时不要删除。
- CoreRobin 只处理你个人文件夹内的普通文件和文件夹。
- 主目录、废纸篓本身、链接、特殊文件和其他磁盘不会被删除。
- 真正删除前，应用会再次核对整棵目标树。发现深层新增文件或其他不一致时会停止操作，并要求重新刷新、确认。
- 删除成功的项目会立即从扇形图和清理篮中消失；失败的项目会保留下来并说明原因。

### 网络

- 网络不再是日常模式的一级页面；选择“网络好像有问题”后，CoreRobin 会先检查当前连接和收发状态，再给出结论。
- 查看当前上传和下载速度、本次启动后的累计流量、网络接口和活动连接。
- 可以按 TCP、UDP 和连接状态筛选列表。
- 系统不一定允许普通应用读取每条连接属于哪个进程，所以没有显示应用名并不代表连接有问题。

### 启动项

- 启动项不再是日常模式的一级页面；选择“开机越来越慢”后才会出现相关检查和可恢复的操作。
- 查看哪些应用会随电脑启动，以及它们来自哪里。
- 支持的第三方项目可以关闭，也可以稍后恢复。
- 系统项目和当前平台无法安全修改的项目只供查看。
- CoreRobin 不会直接删除启动项的配置文件。

### 历史

- 日常模式的“记录”用简单时间线说明电脑什么时候开始变忙、什么时候恢复正常。专业模式保留完整的资源筛选和技术历史。
- 同一次问题的提醒与恢复会合并为一条事件，保留开始时间、持续时长、可能原因和最终结果。
- 你确认执行的应用退出或重启、永久清理和启动项操作也会记录执行状态与复核结果；永久清理只保存项目数量和释放空间，不保存文件路径。
- 记录可以保留 1、7 或 30 天，也可以关闭或随时清空。
- 只有问题持续一段时间时才会提醒，同一问题不会反复弹出。
- 问题稳定恢复后会补充一次恢复提醒；主窗口、状态栏面板和 Robin 伙伴使用同一份健康状态。
- CPU、内存和磁盘提醒可以分别关闭。
- 应用和启动项名称只有在你允许保存名称后才会跨重启保留；不保存命令行、文件路径、文件名或连接地址。

### 设置

- 日常模式的设置提供语言、文字大小、动态效果、桌面提醒、本机历史和 Robin 伙伴偏好。
- 可以让 CoreRobin 在登录系统后静默启动，不打开主窗口；macOS 还可以选择是否在 Dock 和应用切换器中显示图标。
- 专业模式的设置中还可以调整采样速度、连接刷新速度、颜色提示和默认进程视图。

## 状态栏面板

关闭主窗口后，CoreRobin 仍会留在状态栏。单击状态栏图标会打开与主窗口共享同一健康结论的紧凑面板；再次单击会收起，双击会直接打开主窗口。登录时静默启动不会弹出启动画面或主窗口。

选择“Robin 伙伴”会在桌面右下角打开一个紧凑小窗口。它与主窗口、状态栏面板共享当前结论，只显示一个下一步；按 Esc、点击关闭按钮或点到其他窗口都会收起。拖动后的位置会保留，也可以在日常助手设置中选择是否置顶或随应用启动显示。

如果要完全停止监控，请退出 CoreRobin，而不是只关闭窗口。

## 常见问题

### 刚启动时为什么没有网络或磁盘速度？

应用需要先后读取两次数据才能算出速度。等到下一次刷新就会显示。

### 为什么扫描要等很久？

扫描时间主要取决于文件数量和磁盘速度，而不是磁盘总容量。大量小文件通常比少量大文件更慢。页面会持续更新进度，你也可以主动停止。

### 为什么大小和 Finder 或 Explorer 显示的不一样？

CoreRobin 显示文件实际占用的磁盘空间。压缩文件、稀疏文件和硬链接等情况，可能会让这个数字与文件标称大小不同。

### 为什么有些连接没有应用名？

操作系统可能不允许普通应用读取连接属于哪个进程。CoreRobin 不会为了补全这项信息自动申请更高权限。

## 隐私与问题反馈

监控、文件、进程、历史和连接信息只保存在这台电脑上，不会上传或同步。扫描只读取名称、大小、位置等文件信息，不读取文件内容。

普通问题可以在 GitHub Issues 中反馈。请写清楚操作系统、刚才做了什么，以及看到的错误信息。安全问题请使用 GitHub 的私密漏洞报告功能。
