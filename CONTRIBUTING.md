# Contributing

CoreRobin's desktop application source is maintained privately. This repository accepts public bug reports, feature requests, and focused corrections to the public website or user documentation.

## Issues

- Search existing issues before opening a new one.
- Use the provided bug or feature template.
- Keep each issue focused on one reproducible problem or one user need.
- Remove private information before submitting text, screenshots, or diagnostics.

## Website and documentation pull requests

- Keep changes limited to `site/`, `docs/`, or their public build configuration.
- Preserve the Chinese and English versions of user-facing content.
- Run `npm run site:build` before submitting a pull request.
- Do not add application source, internal links, credentials, analytics, or third-party tracking scripts.

By submitting a contribution, you confirm that you have the right to provide it and grant the repository owner permission to use, modify, and distribute it as part of CoreRobin's public website and documentation.
