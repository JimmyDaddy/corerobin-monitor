完整磁盘访问列表里没有 CoreRobin 怎么办

这通常不是权限申请失败，因为 macOS 不一定会自动把应用加入列表。先把 CoreRobin 安装到“应用程序”，再点击列表下方的 + 手动选择它即可。不想授权时，也可以只扫描当前能够访问的位置。

直接答案

先退出从 DMG 打开的 CoreRobin，把 CoreRobin.app 拖到“应用程序”并从那里启动。进入系统设置 → 隐私与安全性 → 完整磁盘访问，点击列表下方的 +，选择“应用程序”里的 CoreRobin.app，打开开关并按系统提示重新打开应用。

按这五步添加

  1. 1

    确认使用的是已安装应用

    不要一直从下载的 DMG 里运行。将 CoreRobin.app 拖到“应用程序”，推出 DMG，再从“应用程序”启动。这样 macOS 才能持续识别同一份应用。

  2. 2

    从 CoreRobin 打开正确设置页

    进入空间清理并选择完整扫描。CoreRobin 会说明用途并打开“隐私与安全性 → 完整磁盘访问”。

  3. 3

    点击 +，不要等待它自动出现

    Apple 的设置页支持通过添加按钮选择应用。点击列表下方的 +，在“应用程序”中选择 CoreRobin.app，再点击“打开”。

  4. 4

    打开开关并完成系统验证

    macOS 可能要求 Touch ID 或管理员密码。如果系统提示退出并重新打开应用，请按提示操作；这项权限通常要在 CoreRobin 重新启动后才会生效。

  5. 5

    返回清理页面确认状态

    重新打开后返回 CoreRobin。应用会重新检查权限并继续；如果仍显示未授权，确认列表中开启的是当前“应用程序”目录里的那一份，而不是旧副本。

Apple 对完整磁盘访问的说明明确写明：可以点击添加按钮、选择应用并打开。 查看 Apple 隐私与安全性指南

添加后仍然不对，检查这些情况

  • “应用程序”中存在多个 CoreRobin 副本，设置里添加的是旧版本或另一个路径。删除不用的副本后重新添加当前应用。
  • 你仍从 DMG、下载目录或开发启动进程运行。退出后只从 /Applications/CoreRobin.app 启动。
  • 开关已打开但应用没有重新启动。完全退出 CoreRobin，而不只是关闭窗口，再重新打开。
  • macOS 显示“应用已损坏”。这不是完整磁盘访问问题；删除旧应用和旧 DMG，重新下载最新版并核对 SHA-256。

暂不授权也能扫描

选择“扫描可访问区域”即可继续。CoreRobin 会扫描当前能够读取的位置，并把受保护或无法读取的位置标记为无法访问，不会猜测这些位置的大小。之后如果需要，可以再授权并重新扫描。

CoreRobin 为什么需要这项权限

macOS 会保护邮件、信息、Safari、Home、Time Machine 备份和其他应用数据。完整磁盘访问让空间扫描能更完整地读取这些位置的文件元数据。CoreRobin 不读取文件内容，不上传扫描结果，也不会因为获得权限就自动删除任何内容。权限由 macOS 管理，你可以随时在同一设置页关闭。
