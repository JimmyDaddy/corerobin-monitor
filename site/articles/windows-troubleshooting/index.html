Windows 电脑变慢与磁盘空间怎么排查

先确认卡顿或空间不足是否持续，再查看任务管理器中的进程、内存和磁盘活动，以及系统盘的可用空间。不要因为一次瞬时高占用就结束进程，也不要在没看清内容前批量清理文件。

直接答案

在问题出现时连续观察 5 到 10 分钟。按 Ctrl + Shift + Esc 打开任务管理器，确认是否一直是同一个应用占用 CPU、内存或磁盘；再进入“设置 → 系统 → 存储”查看系统盘。如果数值很快恢复，先记录这次情况，不要立刻结束进程或删除文件。

按这个顺序排查

  1. 1

    记下问题何时出现

    区分开机后一直很慢、打开某个应用后才慢、游戏或编译时变慢，以及 Windows Update、病毒扫描或云盘同步期间的短时繁忙。只有可以重复出现的现象，才适合继续定位。

  2. 2

    在任务管理器中查看持续占用

    在“进程”页按 CPU、内存或磁盘排序。如果同一进程连续几分钟排在前面，并且卡顿同时存在，它才更可能相关。先保存工作并从应用自身正常退出；不要因为进程名称陌生就强制结束。

  3. 3

    检查内存、磁盘活动和启动应用

    内存接近用满并伴随明显换页、磁盘活动长时间接近满负载，都会让响应变慢。“启动应用”页可以帮助确认是否有不需要的软件随登录启动，但请逐项关闭并观察，不要一次禁用全部。

  4. 4

    确认 C: 盘还有可用空间

    系统更新、临时文件、分页文件和应用缓存都需要系统盘空间。打开“设置 → 系统 → 存储”，先看临时文件、已安装的应用和个人文件分别占了多少，再决定清理对象。

  5. 5

    重启后复现，再检查更新和驱动

    重启可以结束遗留任务，但不能说明根因。如果问题回来，记录复现步骤，再检查最近安装的 Windows 更新、驱动、杀毒软件或常驻工具；一次只改变一项。

用 CoreRobin 怎么看

日常模式

先运行“检查一下”，看问题是否持续，以及 CoreRobin 给出的原因和下一步。短时尖峰不会立即变成待处理事项。

专业模式

需要进一步确认时，再查看进程、CPU、内存、磁盘读写和趋势，并与任务管理器中同一时段的结果互相核对。

Microsoft 也建议使用任务管理器查看 CPU、内存和磁盘使用情况，并检查“性能”和“启动应用”页面。 查看 Microsoft 性能指南

先预览，再清理空间

  • 优先检查下载目录、旧安装包、很久不用的应用、大型视频、虚拟机和回收站。
  • “存储感知”默认主要管理系统盘；启用前先查看临时文件、回收站和下载目录的保留规则。
  • 不要手动删除 Windows、Program Files 或不认识的 AppData 内容。卸载应用应使用“设置 → 应用”。
  • 使用 CoreRobin 清理时，先查看路径和大小；放入清理篮不会立即删除，永久删除前还会重新扫描并要求确认。

Microsoft 说明“存储感知”可以清理临时文件和回收站内容，但下载目录和云端文件是否处理取决于你的设置。 查看 Microsoft 存储指南

Windows 版目前是早期预览

Windows 安装包已经通过自动构建和启动检查，但实机覆盖仍少于 macOS。温度、电池健康度、进程归属和部分启动项信息会受到硬件、驱动和系统接口影响；没有可靠数据时，CoreRobin 会显示“不可用”，不会用估算值替代。
