Finder 与磁盘扫描结果为什么不同

多数情况下不是谁算错了，而是这些数字表示的含义不同：文件内容有多大、实际占用了多少磁盘空间，以及整个磁盘还能写入多少数据，并不是一回事。

先说结论三个数字常见原因CoreRobin 算法该信哪个
直接答案

查看磁盘还能用多少空间，以 macOS 显示的“可用”为准；定位哪些路径实际占用磁盘，用 CoreRobin 的空间地图；查看单个文件的标称内容大小，用 Finder。比较前确保是同一路径、同一时刻，并让扫描完成。

先分清三个数字

逻辑大小

文件包含的数据长度。稀疏文件可能声称很大，但没有为所有空白区域分配磁盘块。

实际占用

文件系统为内容分配的磁盘块。压缩、稀疏和共享数据会让它与逻辑大小不同。

磁盘可用空间

整个卷当前还能提供的空间，可能包含系统能够按需回收的内容，不能靠把目录大小简单相加得到。

最常见的六个差异来源

  1. 1

    压缩文件和稀疏文件

    内容长度和已分配磁盘块不相等。虚拟机磁盘、数据库和某些缓存尤其常见。

  2. 2

    硬链接和共享内容

    多个路径可能指向磁盘上的同一份数据。如果把每条路径都相加，就会重复计算，因此扫描时需要识别并去重。

  3. 3

    APFS 克隆、快照和可清理空间

    APFS 快照、可清理的系统数据和多个文件共用的存储空间，都会影响 macOS 显示的可用空间，但它们不一定会作为普通文件出现在扫描结果中。

  4. 4

    iCloud 和其他云端占位文件

    Finder 可以显示云端项目的逻辑大小，但文件内容可能没有完整下载。CoreRobin 会跳过由云盘应用管理的根目录，避免扫描时触发大量下载。

  5. 5

    权限和不可访问位置

    未授予完整磁盘访问时，邮件、信息和其他应用数据可能无法扫描。CoreRobin 会把这些位置标记为无法访问，不会用估算值填补。

  6. 6

    扫描时文件仍在变化

    浏览器缓存、日志、下载和虚拟机磁盘可以边扫描边变化。Finder、系统设置和 CoreRobin 如果不是同一时刻取样，结果自然不会完全一致。

CoreRobin 怎样计算

  • 在 macOS 和 Linux 上按文件系统报告的已分配块计算实际占用，而不是只用文件长度。
  • 同一次扫描中会识别指向同一份文件的硬链接，避免重复计算。
  • 不跟随符号链接，不跨到其他磁盘，避免循环、越界和把外置盘计入系统盘。
  • 扫描结果只代表完成时的状态。删除前会重新检查路径和文件是否发生变化，发现不一致就会取消操作。

到底该信哪个数字

想知道还能存多少看 macOS 的可用空间
想找到占空间的路径看 CoreRobin 完成后的空间地图
想知道单个文件内容大小看 Finder 的大小信息
想解释总数差异确认路径、扫描时间和权限一致，再检查共享或云端内容

不要为了让两个界面的总数完全相同，就去删除不认识的文件。只清理你知道用途、并且确定不再需要的内容；完成后查看 macOS 显示的可用空间是否增加。
